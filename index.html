git流程操作以及git补救操作。
1.git init
创建一个git本地库，现在暂时是空的。
2.vi .gitignore
在项目根目录下创建一个git的忽略目录。
不然git就是 你写啥，敢传啥，pyc一大把。说多了都是泪呀。
2.git add
如果你没有创gitignore的前车之鉴，那么恭喜你。
你可能要翻车了。😄喜闻乐见。
然而善(翻)良(车)如(无)我(数)，git rm -r –cached,这个可以将之前add的文件删除，重新变为untracked。
3.gst
看看现在库里的状态，有哪些添加了的。那些改变的 blablabla的。。
4.git diff
不放心的话，可以用这个命令 看到具体的语句的更改
反正从来对自己保持怀疑态度
杜绝自己走上 从删库到跑路的 不归路
5.git commit -m ‘bbb’
提交到本地代码库
如果提交了，错了 呵呵，很正常。
重新修改后
git commit -amend
只会产生一次提交。
6.git add origin ‘url’
7.git push origin

提交代码的时候，一定要确认没有问题的话再提交。要淡定。
合作的话，先从远端pull代码下来，然后merge，有冲突解决冲突（解决冲突不要改远端的代码，可以对照着github上面调整自己的代码）。然后再进行代码的编写，然后重复2-7

遇到坑，再更，i will keep following。

③反爬虫机制ALL

Posted on 2017-02-04   |   更新于 2017-02-04

爬虫与反爬虫

爬虫(Spider)，反爬虫(Anti-Spider)，反反爬虫(Anti-Anti-Spider)，这之间的斗争恢宏壮阔...^_^很有意思的一句话，但是八九不离十。但是众所周知结论已经注定了，在爬虫与反爬虫的对弈中，爬虫一定会胜利。作为胜利的一方哈哈，留下只言片语啦。这是本系列记录的第三篇文章，主要是讨论如何防止ban、绕过网站的防御措施尽量多的爬取数据。

策略

主要思路是项目setting设置+使用网上免费提供的大陆高匿IP代理.实现当IP被禁止时自动切换新的高匿IP的代理。即在spider里检查是否被ban, 如果被ban了则重新yield一个请求, 并设置meta[“change_proxy”]=True, 然后在middleware里的process_request处检查这个属性, 如果为True则更换代理.还有个问题就是代理数量不足的问题. 由于免费代理一般都是临时性的, 随着运行时间的增长, 有些代理会失效, 这样到最后就没有可用的代理了. 于是我在新的ProxyMiddleware里, 每次换新代理时检查代理列表中有效代理的数量, 如果小于一个阈值, 则从网上抓新的免费代理扩充代理列表.

放慢抓取速度

减小对于目标网站造成的压力。

禁用cookies

setting文件中设置COOKIES_ENABLED=False

设置延迟下载

setting文件中设置DOWNLOAD_DELAY=10

添加更多的USER_AGENTS

突破反爬虫机制继续高频率抓取

在网络爬虫抓取信息的过程中，如果抓取频率高过了网站的设置阀值，将会被禁止访问。通常，网站的反爬虫机制都是依据IP来标识爬虫的。
通过使用高匿IP突破反爬虫机制继续高频率抓取。高匿名代理不改变客户机的请求，这样在服务器看来就像有个真正的客户浏览器在访问它，这时客户的真实IP是隐藏的，服务器端不会认为我们使用了代理。但是这样需要多个稳定的代理IP。有了多个稳定的IP后，在高过了网站的设置阀值被禁止访问时可以自动切换IP绕过网站防御机制，继续进行爬取。
