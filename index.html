进程、线程 -- flag篇

Posted on 2017-03-25   |   更新于 2017-03-26

  立一个新的flag，也算为自己开一个新坑。打算由浅入深的比较系统的学习一下进程和线程的相关知识。这是该系列的第一篇，主要介绍概念性的东西，以后慢慢的（也许快快的~）会加入更经典的关于进程、线程处理的代码。
    进程 – 程序的一次执行
    线程 – CPU调度的最小单位
  看到这里，这个定义下的其实我有点奇怪。根据这个定义的话，那么网上关于mutilprocessing的很多的例子看起来就比较的没有内涵。

这代码写的, 基本上就是只有告诉我os.getpid能获取pid的作用了。
这个完全换成线程也一样好不好。

进程和线程的主要功能，是提高cpu的利用的效率，缩短程序运行的时间。
  每个进程，应该负责一个较为独立的功能或者根本就是一个独立的程序的。进程是有自己的独立的区域的，进程间需要通信的话，一般使用Queue或者Pipes（Queue是进程、线程安全的。Pipes双工通信。一个安全，一个快。）。Chrome就是一个典型的多进程程序，里面每个标签页、扩展、插件都是单独的进程，各自独占资源，相互隔离，一个进程出错死掉只会影响一个页面或者插件，再也不会出现Flash插件出错崩溃导致整个浏览器崩溃的情况了。
  线程的话，一般多个线程被包含在一个进程里，cpu调度的基本单位是线程，所以拥有多个线程的话，就近似于程序跑在多个cpu上，并且程序的各个线程都在并行的执行。线程的资源是共享的，父进程的资源由所有线程共享，但是各个线程可以独自拥有自己的变量。由于线程间共享进程中的全局变量，所以当其他线程改变了共享的变量时，可能会对本线程产生影响。所谓线程安全的约束是指一个函数被多个并发线程反复调用时，要一直产生正确的结果。要保证线程安全，主要是通过加锁的方式保证共享变量的正确访问。

垃圾回收机制
  Python 采用垃圾回收机制来清理不再使用的对象；Python 提供gc模块释放不再使用的对象，Python 采用‘引用计数’ 的算法方式来处理回收，即：当某个对象在其作用域内不再被其他对象引用的时候，Python 就自动清除对象；Python 的函数collect()可以一次性收集所有待处理的对象(gc.collect())

git使用技巧之大壹

Posted on 2017-03-25   |   更新于 2017-03-25

git流程操作以及git补救操作。是初零的续篇。
1.git init
创建一个git本地库，现在暂时是空的。
2.vi .gitignore
在项目根目录下创建一个git的忽略目录。
不然git就是 你写啥，敢传啥，pyc一大把。说多了都是泪呀。
2.git add
如果你没有创gitignore的前车之鉴，那么恭喜你。
你可能要翻车了。😄喜闻乐见。
然而善(翻)良(车)如(无)我(数)，git rm -r –cached,这个可以将之前add的文件删除，重新变为untracked。
3.gst
看看现在库里的状态，有哪些添加了的。那些改变的 blablabla的。。
4.git diff (HEAD)
不放心的话，可以用这个命令 看到具体的语句的更改
反正从来对自己保持怀疑态度
杜绝自己走上 从删库到跑路的 不归路
5.git commit -m ‘bbb’
提交到本地代码库
如果提交了，错了 呵呵，很正常。
重新修改后
git commit -amend
只会产生一次提交。
6.git add origin ‘url’
7.git push origin

提交代码的时候，一定要确认没有问题的话再提交。要淡定。
合作的话，先从远端pull代码下来，然后merge，有冲突解决冲突（解决冲突不要改远端的代码，可以对照着github上面调整自己的代码）。然后再进行代码的编写，然后重复2-7

爬虫与反爬虫

爬虫(Spider)，反爬虫(Anti-Spider)，反反爬虫(Anti-Anti-Spider)，这之间的斗争恢宏壮阔...^_^很有意思的一句话，但是八九不离十。但是众所周知结论已经注定了，在爬虫与反爬虫的对弈中，爬虫一定会胜利。作为胜利的一方哈哈，留下只言片语啦。这是本系列记录的第三篇文章，主要是讨论如何防止ban、绕过网站的防御措施尽量多的爬取数据。

策略

主要思路是项目setting设置+使用网上免费提供的大陆高匿IP代理.实现当IP被禁止时自动切换新的高匿IP的代理。即在spider里检查是否被ban, 如果被ban了则重新yield一个请求, 并设置meta[“change_proxy”]=True, 然后在middleware里的process_request处检查这个属性, 如果为True则更换代理.还有个问题就是代理数量不足的问题. 由于免费代理一般都是临时性的, 随着运行时间的增长, 有些代理会失效, 这样到最后就没有可用的代理了. 于是我在新的ProxyMiddleware里, 每次换新代理时检查代理列表中有效代理的数量, 如果小于一个阈值, 则从网上抓新的免费代理扩充代理列表.

放慢抓取速度

减小对于目标网站造成的压力。

禁用cookies

setting文件中设置COOKIES_ENABLED=False

设置延迟下载

setting文件中设置DOWNLOAD_DELAY=10

添加更多的USER_AGENTS

突破反爬虫机制继续高频率抓取

在网络爬虫抓取信息的过程中，如果抓取频率高过了网站的设置阀值，将会被禁止访问。通常，网站的反爬虫机制都是依据IP来标识爬虫的。
通过使用高匿IP突破反爬虫机制继续高频率抓取。高匿名代理不改变客户机的请求，这样在服务器看来就像有个真正的客户浏览器在访问它，这时客户的真实IP是隐藏的，服务器端不会认为我们使用了代理。但是这样需要多个稳定的代理IP。有了多个稳定的IP后，在高过了网站的设置阀值被禁止访问时可以自动切换IP绕过网站防御机制，继续进行爬取。
